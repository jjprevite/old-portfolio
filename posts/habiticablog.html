Contributing to Habitica

June 11th, 2017

Description

I contributed to an open-source project called Habitica, which is a “gamified task-manager.” Basically, it’s a to do list with features you might find in a typical RPG game. It’s a great tool for creating good habits and making life feel like a game. It’s a lot of fun, especially if you have friends that are into it as well.

For my contribution, I worked with a friend on solving this@ issue. The main problems of the issue were the following: -the Group Plans Task Approval List page didn’t indicate the task type that needed to be approved -it didn’t render markdown or emojis in the title or description

To fix this, we had to modify a few files- mainly a JSON, and two files written in Javascript.

Tecnologies Used

-MongoDB, Express, Angular, Node and Jade(now Pug).

What I struggled with

Finding the files that had the lines of code that needed to be modified for the first issue (task type) and understanding enough Jade to be able to render markdown and emojis based on the nom packages used in the project.

The issue took two weeks to finish, or around 14-18 hours to fix. The majority of that time was knowing which files to look in.

To resolve this issue, I spent hours looking through the project folder, searching different terms like “group” and “group approvals” until I was able to locate the lines of code to fix. After I found the files, I faced a new challenge of figuring out how to take data from the database and be able to call it in the JSON file to be displayed on the Group Plans Task Approval page.

I looked at the objects in the database and found a task type key-value pair with the information I needed. Then I had to it to an object as a property on an object in a file called groupApprovalsController.js so that it could be referenced in the JSON file.

To figure all this out with my friend, we used our Javascript knowledge and did a lot of testing different things out until we were able to find a solution that worked.

The next challenge we faced was being able to render markdown and emojis. Luckily, the project uses an npm package developed by one of the main project developers and the docs were easy to follow. We looked at the similar Jade pages and recognized patterns in how the markdown was render.

Once we knew the syntax for markdown and the page we had to add the markdown to, we thought the rest would be easy. But it wasn’t. We struggled to figure out how to pass in the specific text we needed. After hours of frustration and many unsuccessful attempts, we finally figured it out. We had to pass in a function called “approvalTitle” with an argument of “approval” and set text = to that so the line we ended up writing was markdown(text = “approvalTitle(approval)”).

Of course we thought we were done till we noticed the new code shifted the location of the “Approve” button on the page. One solution opened up a new problem but we continued on. After digging around, Rachel, my friend, was able to find a Bootstrap fix that made it look a little better. We asked for help in a chat forum for Habitica open-source developers and were advised to submit a pull request to receive feedback.

I submitted one where another contributor made a few suggestions on how to fix the formatting issue we were running into. After cleaning up my commits and organizing them into meaningful ways so that other developers would understand them, I submitted a new pull request which has been approved and will soon be merged!

What I learned

Overall, I learned a lot from this project! The main things were:

  • how to work on code written by other people
  • how to work with another developer to fix a bug
  • how to write meaningful commit messages
  • how to submit a pull request
  • how to contribute to an open-source project

It was a neat experience and I feel thankful for it to have been so positive my first open-source project. I hope to continue contributing to Habitica in the future and other open-source projects that come my way.
